## پہلے کبھی استعمال نہیں کیا؟ یہاں سے شروع کریں۔

اگر آپ نے Obsidian کے بارے میں کبھی نہیں سنا یا Markdown نوٹ نہیں لکھا، فکر نہ کریں۔ ایک پیراگراف میں:

**Obsidian** ایک مفت ایپ ہے ([obsidian.md](https://obsidian.md)) جو آپ کے کمپیوٹر پر `.md` فائلز (Markdown، سادہ متن) کا فولڈر پڑھتی ہے اور انہیں ایک خوبصورت، باہم جڑی ہوئی نوٹ بک کے طور پر دکھاتی ہے۔ چونکہ سب کچھ سادہ متن ہے، **آپ کے نوٹس ہمیشہ آپ کے ہیں** — کوئی کلاؤڈ لاک ان نہیں، کوئی ادا شدہ سبسکرپشن نہیں، کوئی ایسا وینڈر نہیں جو غائب ہو سکے۔ Mind++ Obsidian کے ڈھانچے کو بنیاد کے طور پر استعمال کرتا ہے کیونکہ یہ آج موجود ذاتی علم کے لیے سب سے پائیدار اور نجی فارمیٹ ہے۔

**Claude Cowork** Claude کی ڈیسک ٹاپ ایپ ہے ([claude.ai/download](https://claude.ai/download)) جو آپ کے کمپیوٹر پر فائلز پڑھ اور لکھ سکتی ہے۔ جب آپ Mind++ انسٹال کرتے ہیں، Claude اس فولڈر میں آپ کے نوٹس کو منظم کرنے کا ایک مخصوص طریقہ سیکھ لیتا ہے — تاکہ جب آپ کہیں "میری احمد سے میٹنگ ہوئی"، وہ نوٹ لکھ دے، ایکشن آئٹمز نکال لے، اور آپ کی ٹاسک لسٹ خود بخود اپ ڈیٹ کر دے۔

**آپ کو ڈویلپر ہونے کی ضرورت نہیں۔** Git جاننے کی ضرورت نہیں۔ YAML جاننے کی ضرورت نہیں۔ Mind++ ڈھانچہ سنبھالتا ہے؛ آپ بس Claude سے بات کرتے ہیں۔

## Mind++ کیوں؟

- **کوئی وینڈر لاک ان نہیں** — آپ کا ڈیٹا سادہ `.md` فائلز ہیں، کہیں بھی کسی بھی ایڈیٹر میں پڑھی جا سکتی ہیں
- **ڈیفالٹ رازداری** — سب کچھ آپ کے فولڈر میں، آپ کی مشین پر رہتا ہے
- **Obsidian نیٹیو** — گراف ویو، بیک لنکس، سرچ اور ٹیگز باکس سے باہر کام کرتے ہیں
- **AI پروسیس کرتا ہے، انسان تصدیق کرتا ہے** — Claude پروسیس کرتا ہے؛ کسی بھی منتقلی یا حذف سے پہلے آپ تصدیق کرتے ہیں
- **فعال، ردعمل نہیں** — تاخیر شدہ کاموں اور خاموش کلائنٹس کو پوچھے بغیر سامنے لاتا ہے
- **کثیر لسانی** — پلگ ان جس زبان میں آپ لکھتے ہیں اس میں کام کرتا ہے؛ Claude آپ کے ساتھ ڈھل جاتا ہے

## 5 منٹ میں تیز آغاز

> 👉 **تفصیلی مرحلہ وار گائیڈ چاہتے ہیں؟** → [Wiki پر انسٹالیشن گائیڈ](https://github.com/felipe-venturini/mind-plus-plus/wiki/Installation-Guide)

### 1. تین اوزار انسٹال کریں

| اوزار | یہ کیا کرتا ہے | لنک |
|-------|---------------|-----|
| **Claude Cowork** (لازمی) | Claude ایپ جو پلگ ان چلاتی ہے | [claude.ai/download](https://claude.ai/download) |
| **Obsidian** (انتہائی سفارش کردہ) | آپ کے نوٹس کے لیے بصری نیویگیٹر | [obsidian.md](https://obsidian.md) |
| **آپ کا سنک انتخاب** (اختیاری) | بیک اپ اور کہیں بھی رسائی — ایک منتخب کریں: [Obsidian Git](https://github.com/Vinzent03/obsidian-git)، Google Drive، iCloud Drive، OneDrive یا Dropbox | دیکھیں [Sync Strategies](https://github.com/felipe-venturini/mind-plus-plus/wiki/Sync-Strategies) |

### 2. منتخب کریں کہ آپ کا والٹ کہاں رہے گا

اپنے کمپیوٹر پر ایک فولڈر منتخب کریں۔ یہ وہ جگہ ہے جہاں ہر نوٹ، میٹنگ اور فیصلہ محفوظ ہوگا۔

- **سب سے آسان** — `~/Documents/Mind++` جیسا فولڈر بنائیں
- **خودکار طور پر کلاؤڈ میں بیک اپ** — کلاؤڈ سے مطابقت پذیر جگہ کے اندر فولڈر بنائیں: `~/Google Drive/Mind++`، `~/OneDrive/Mind++`، `~/Dropbox/Mind++` یا `~/Library/Mobile Documents/com~apple~CloudDocs/Mind++` (iCloud)۔ Obsidian اور Cowork وہی فائلیں پڑھیں گے، اور کلاؤڈ سروس انہیں ورژن کرے گی اور آپ کے تمام آلات پر مطابقت پذیر بنائے گی۔
- **پاور یوزر کا راستہ** — اپنے والٹ کو ایک نجی GitHub ریپو میں ورژن کرنے کے لیے [Obsidian Git پلگ ان](https://github.com/Vinzent03/obsidian-git) استعمال کریں (کمٹ ہسٹری، رول بیک اور Obsidian موبائل ایپ کے ذریعے Android/iOS سنک فراہم کرتا ہے)

> ٹریڈ آف اور ہر استعمال کیس کے لیے سفارش کے لیے [Wiki پر Sync Strategies کا صفحہ](https://github.com/felipe-venturini/mind-plus-plus/wiki/Sync-Strategies) دیکھیں۔

### 3. Mind++ پلگ ان انسٹال کریں

1. تازہ ترین [`mind-plus-plus.plugin`](https://github.com/felipe-venturini/mind-plus-plus/releases/latest) فائل ڈاؤنلوڈ کریں
2. Claude Cowork کھولیں → **Plugins**
3. `.plugin` فائل کو پلگ ان ایریا میں گھسیٹیں (یا انسٹال پر کلک کریں)

### 4. Cowork میں اپنا والٹ فولڈر منتخب کریں

Claude Cowork میں، مرحلہ 2 میں منتخب کردہ فولڈر کو اپنے ورک اسپیس کے طور پر منتخب کریں۔ Claude براہ راست اس میں پڑھے گا اور لکھے گا۔

### 5. سیٹ اپ چلائیں

Claude کو بتائیں:

> **"میرا دوسرا دماغ سیٹ اپ کریں"**

سیٹ اپ آپ کے کام کے سیاق و سباق کے بارے میں پوچھے گا (کلائنٹس، پروڈکٹس، اوزار)، فولڈر ڈھانچہ بنائے گا، ایک ذاتی نوعیت کا `CLAUDE.md` لکھے گا، اور روزانہ بریف اور ہفتہ وار جائزہ شیڈول کرنے کی پیشکش کرے گا۔

### 6. استعمال شروع کریں

- ہر صبح **"brief"** کہیں → Claude آج کا ایجنڈا، تاخیر شدہ ٹاسک اور فالو اپ دکھائے گا
- ہر کال کے بعد **"میری X سے میٹنگ ہوئی"** کہیں → Claude نوٹ لکھے گا اور آپ کے ٹاسک اپ ڈیٹ کرے گا
- ہر جمعہ **"weekly review"** کہیں → Claude ہفتے کو مضبوط کرے گا

بس۔ آپ جتنا زیادہ استعمال کریں گے، آپ کا والٹ اتنا ہی ایک حقیقی دوسرا دماغ بن جائے گا۔

## پلگ ان میں کیا ہے

### 12 skills

| Skill | یہ کیا کرتی ہے | مثال ٹرگر |
|-------|---------------|-----------|
| `setup` | صفر سے والٹ کا بوٹ اسٹریپ | "میرا دوسرا دماغ سیٹ اپ کریں" |
| `user-profile` | آپ کا پیشہ ورانہ اور ذاتی پروفائل محفوظ کرتا ہے | "میرا پروفائل اپ ڈیٹ کریں" |
| `new-meeting` | ٹرانسکرپٹس یا نوٹس کو ایکشن کے ساتھ منظم میٹنگ ریکارڈ میں بدلتا ہے | "میری X سے میٹنگ ہوئی" |
| `meeting-prep` | میٹنگ سے پہلے بریفنگ — حالیہ میٹنگز، کھلے ٹاسک، گفتگو کے نکات | "X کے ساتھ میٹنگ کی تیاری" |
| `capture-idea` | کم رگڑ کے ساتھ فوری آئیڈیا کیپچر، خودکار درجہ بندی | "نوٹ کریں: …" |
| `new-decision` | درست اسکوپ میں ایک آرکیٹیکچر فیصلہ ریکارڈ (ADR) درج کرتا ہے | "ایک فیصلہ ریکارڈ کریں" |
| `knowledge-search` | ترکیب اور ماخذ لنکس کے ساتھ والٹ میں سیمانٹک سرچ | "میں X کے بارے میں کیا جانتا ہوں" |
| `stakeholder-update` | کلائنٹ، پروڈکٹ یا پروجیکٹ کے لیے بیرونی اسٹیٹس رپورٹ | "X پر اسٹیٹس اپ ڈیٹ" |
| `daily-brief` | صبح کی بریفنگ: ایجنڈا، تاخیر شدہ ٹاسک، فالو اپ | "brief" |
| `weekly-review` | جمعہ کی مضبوطی: کیا کیا گیا، کیا خطرے میں ہے، کیا آنے والا ہے | "weekly review" |
| `process-inbox` | بے گھر فائلوں کی چھانٹی، منزل کی تجویز، تصدیق کا انتظار | "inbox پروسیس کریں" |
| `process-meeting-emails` | خودکار پائپ لائن: Gemini / Fireflies / Otter ای میلز → منظم نوٹس | "میٹنگ پائپ لائن چلائیں" |

### 2 agents

ایجنٹس خود مختار ذیلی عمل ہیں جو الگ تھلگ سیاق و سباق میں چلتے ہیں اور صاف نتیجہ واپس کرتے ہیں۔

| Agent | یہ کیا کرتا ہے | کب چلتا ہے |
|-------|---------------|------------|
| `vault-researcher` | بنیادی گفتگو کو اوور لوڈ کیے بغیر کئی فائلوں میں گہری قراءت | خودکار طور پر، جب `knowledge-search` >15 فائلیں میچ کرے |
| `vault-auditor` | مکمل والٹ ہیلتھ چیک: تاخیر شدہ ٹاسک، خاموش کلائنٹس، یتیم فائلیں، پرانے فیصلے | مطالبہ پر ("میرا والٹ آڈٹ کریں") یا ہفتہ وار شیڈول |

### شیڈولڈ ٹاسکس

Cowork کسی بھی skill کو شیڈول پر چلا سکتا ہے۔ سیٹ اپ یہ بنانے کی پیشکش کرتا ہے:

| ٹاسک | ڈیفالٹ شیڈول | Skill |
|------|--------------|-------|
| روزانہ بریف | صبح 8:00، کام کے دن | `daily-brief` |
| ہفتہ وار جائزہ | شام 5:00، جمعہ | `weekly-review` |
| ای میل پائپ لائن | ہر 2 گھنٹے میں | `process-meeting-emails` |
| والٹ آڈٹ | صبح 9:00، پیر | ایجنٹ `vault-auditor` |

آپ بعد میں شیڈول بنا، ترمیم یا حذف کر سکتے ہیں — بس Claude سے کہیں۔

## آپ کا والٹ کیسے منظم ہوتا ہے

Mind++ یہ ڈھانچہ بناتا اور برقرار رکھتا ہے۔ آپ کو اسے یاد رکھنے کی ضرورت نہیں — Claude جانتا ہے کہ ہر چیز کہاں جاتی ہے۔

```
clients/{کلائنٹ}/       ← ہر کلائنٹ کے لیے ایک فولڈر
  {Client}.md           ← رابطے، فعال پروجیکٹس، اسٹیٹس کے ساتھ ڈیش بورڈ
  meetings/             ← YYYY-MM-DD عنوان.md
  decisions/            ← ADRs — NNN عنوان.md
  references/           ← بریفنگز، اسپریڈ شیٹس، معاون دستاویزات
  Tasks {Client}.md     ← کھلے ٹاسک کا ٹریکر

products/{پروڈکٹ}/      ← آپ کی اپنی پروڈکٹس (اگر قابل اطلاق ہو)
operational/            ← اندرونی کام، ٹیم، کنفگ
personal/               ← ذاتی پروجیکٹس، اہداف، نوٹس
shared/                 ← ٹیمپلیٹس، پرامپٹس، منسلکات
inbox/                  ← غیر یقینی آئٹمز — process-inbox کے ساتھ چھانٹی
archive/                ← بند آئٹمز — یہاں منتقل، کبھی حذف نہیں
```

مزید تفصیل [Wiki](https://github.com/felipe-venturini/mind-plus-plus/wiki) پر۔

## رازداری

سب کچھ آپ کے کمپیوٹر پر رہتا ہے۔ Mind++ آپ کا ڈیٹا کہیں نہیں بھیجتا۔ واحد بیرونی سروسز وہ ہیں جنہیں آپ Cowork میں صراحت سے منسلک کرتے ہیں (مثلاً، میٹنگ پائپ لائن کے لیے Gmail، روزانہ بریف کے لیے Google Calendar) — اور یہ آپٹ ان ہیں۔

اگر آپ Obsidian Git یا کلاؤڈ سے مطابقت پذیر فولڈر (Drive، OneDrive، Dropbox، iCloud) استعمال کرتے ہیں، تو آپ کے نوٹس اس سروس پر نقل ہوتے ہیں — لیکن آپ نے اسے منتخب کیا ہے اور آپ اسے کنٹرول کرتے ہیں۔

آپ کا یوزر پروفائل اور یادیں والٹ کے اندر ایک پوشیدہ `.auto-memory/` فولڈر میں مقامی طور پر رہتی ہیں۔

## اکثر پوچھے گئے سوالات

مختصر فہرست — مکمل FAQ [Wiki](https://github.com/felipe-venturini/mind-plus-plus/wiki/FAQ) پر۔

**کیا مجھے Markdown جاننے کی ضرورت ہے؟** نہیں۔ Claude آپ کے لیے Markdown لکھے گا۔ آپ قدرتی زبان میں بات کرتے ہیں۔

**کیا میں Obsidian کے بغیر استعمال کر سکتا ہوں؟** ہاں — پلگ ان کو صرف فولڈر کی ضرورت ہے۔ Obsidian کی انتہائی سفارش کی جاتی ہے کیونکہ گراف ویو اور بیک لنکس والٹ کو قابل نیویگیشن بناتے ہیں، لیکن تکنیکی طور پر آپ فائلوں کو کسی بھی ایڈیٹر میں کھول سکتے ہیں۔

**کیا میں انگریزی کے علاوہ کسی اور زبان میں استعمال کر سکتا ہوں؟** ہاں۔ سیٹ اپ کے وقت (یا کسی بھی وقت) Claude کو اپنی پسندیدہ زبان بتائیں، اور وہ اس زبان میں نوٹس، بریف اور جائزے لکھے گا۔ پلگ ان کا فولڈر ڈھانچہ ڈیزائن کے لحاظ سے انگریزی نام استعمال کرتا ہے (پورٹیبیلٹی کے لیے)، لیکن آپ کا مواد آپ کی زبان میں ہے۔

**اگر میرے پاس پہلے سے Obsidian میں نوٹس ہیں تو کیا ہوگا؟** Mind++ کو اپنے موجودہ والٹ کی طرف اشارہ کریں۔ سیٹ اپ موجودہ فائلوں کا پتہ لگاتا ہے اور ضم کرتا ہے، اوور رائٹ نہیں کرتا۔

## حصہ لیں

Mind++ MIT لائسنس کے تحت اوپن سورس ہے۔ شراکت کا خیر مقدم ہے۔

- **بگ رپورٹس اور فیچر درخواستیں** → [ایک issue کھولیں](https://github.com/felipe-venturini/mind-plus-plus/issues)
- **کوڈ کی شراکت** → دیکھیں [CONTRIBUTING.md](CONTRIBUTING.md)
- **ترجمے** → ایک `README.{زبان}.md` فائل شامل کریں اور ایک PR کھولیں
- **نئی skills** → `skills/*/SKILL.md` میں فارمیٹ کی پیروی کریں اور ایک PR کھولیں
